PostgreSQL 第一章 数据库集群、数据库和表

Posted on 2020-10-20

PostgreSQL 数据库集群、数据库和表，表对应的文件的Page存储结构。以及读写流程。

1.1 数据库集群的逻辑结构

数据库集群包含了许多数据库，每个数据库包含的很多 Table，Table 之间互相独立。

每一个数据库用对象标识符(OID)标识。

1.2 数据库集群的物理结构

根目录 PGDATA。

表空间 与其他 RDBMS 不同，包含了 base 目录之外的其他数据。

Base 目录是数据库目录下的一个基本目录，每个 Table数据 和 它的索引 至少是一个文件。

1.2.1 数据库集群文件结构

根目录 PGDATA

1.2.2 数据库文件结构

数据库: PGDATA/base/DataBase/。

1.2.3 数据库表文件结构

Table: PGDATA/base/DataBase/Table，其中 Table 文件包含了 Data 和 Index。

如果 Table 大于 1G: Table: PGDATA/base/DataBase/Table.1，即会创建 Table(所对应的id).1 文件追加新的数据，填满后以此类推。

1.3 堆表文件内存结构

堆文件(heap Table 和 index) 被分成了许多 Page(或者Block，本质是固定字节数的存储空间大小)，Page 从0开始编号。如果堆文件中的所有Page全被填满，会在文件中追加Page。

每个Page内存的存储结构 由 文件类型决定。

Head Table File 中 Page 结构

  1. heap tuple(s): 存放数据。值得注意的是 Tuple 是以 Page 的末尾为开始，即第一个 Tuple 在 Page 的最末尾。
  2. line pointer(s): 存放了 heap tuple(s) 的索引。pointers 形成了一个数组。
  3. header data
    • pd_lsn: 记录了这个 Page 最后修改的时间(不一定是时间，可能只是一个编号)。
    • pd_checksum: CheckSum value of this Page。用途？
    • pd_lower, pd_upper: ?
    • pd_special: 在 Table(?) 中的 Page的 pd_special 指向 Page 的末尾。在 index 的 Page 中指向的是 特殊区域(是仅仅拥有index 和 特殊数据(表示索引的类型)) 的开始，。

Page File 和 Index File 是分开的。

1.4 写和读 Tuple 的方法

1.4.1 写 Heap Tuple

  1. 按顺序存放到上一个 Tuple 后面。
  2. 生成一个新的 line pointer，指向新的 Tuple。
  3. 修改 Head Info 中其他字段的信息。

1.4.2 读 Heap Tuple

方法一: 顺序读，扫描所有 Page。

方法二: 在 Index File 查找，根据查询数据在索引数据结构找到对应的 Tuple 的 TID((block = 7, Offset = 2)，根据 TID 可以直接找到对应 Page 和 第几个 Tuple。

PostgreSQL 也支持根据 TID 直接读 Tuple。
